# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- `ritual_quick_patch.py` — automated patching tool for quick deployment of
  hybrid provider patches to ritual installations; supports ZIP and directory
  targets
- `patch_files/` directory — container for patch assets used by
  `ritual_quick_patch.py`
- `mcm-portal/run_mcm_portal.sh` — cross-platform shell launch script
  generated by `setup_mcm_portal.py`
- `.gitignore` entries for `mcm-portal/` and `patch_output/`

### Fixed
- Removed hardcoded Windows path
  (`c:\Users\lord ones\Documents\GitHub\ritual`) from `setup_mcm_portal.py`;
  the script now derives its root from `__file__` and works on any system
- Removed unused `shutil` import in `setup_mcm_portal.py`

### Changed
- `setup_mcm_portal.py` generates a `run_mcm_portal.sh` shell script in
  addition to the existing `RUN_MCM_PORTAL.bat`, providing cross-platform
  support for Linux and macOS users
- `setup_mcm_portal.py` updated docstring and module header to reflect its
  purpose accurately

## [1.0.0] - 2026-01-01

### Added
- Initial release of RITUAL
- Hermetic/Mystical themed UI
- LLM Provider discovery (LM Studio, MSTY)
- MCM (Model Context Management) file management
- Encrypted API key storage (Sigils)
- Dashboard with quick actions
- Grimoire for context file management
- RESTful API backend
- FastAPI server with uvicorn
- One-button interface design

### Features
- Beautiful 60fps animations
- Dark theme with purple accent colors
- Local-only design (no cloud dependencies)
- API key encryption using Fernet
- Real-time provider status checking
